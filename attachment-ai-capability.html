01 · Overview

这不是用 AI 写代码，
而是用 AI 操作整台电脑

本案例记录了在一次连续对话中，以纯自然语言指令驱动 Claude AI，完成了摄像头拍照录像、Telegram Bot 全栈部署、系统服务自启动等一系列任务——无需手动编写或执行任何代码。

核心洞察：Claude MCP（Model Context Protocol）将 AI 的能力边界从「文本生成」扩展到了「物理世界执行」。AI 在这里扮演的角色是系统操作员，而非代码建议者。

问题自愈 报错后提示修复方向 主动诊断代理 / 权限 / API 问题并自动切换

03 · What Was Built

单次对话完成的实战项目

多媒体硬件控制

摄像头 · 录像 · 语音
  • 通过 tccutil 重置系统权限 + AppleScript 绕过 MCP 进程权限限制，实现摄像头实时拍照
  • 调用 ffmpeg avfoundation 录制 1280×720 视频，直接推送至 Telegram
  • 调用 macOS say 合成中文 TTS 语音（.m4a），通过系统扬声器播放
  • 自动诊断帧率不兼容问题（30fps → 29.97fps），无需人工介入

Telegram Bot 全栈部署

从 0 到生产上线（包含故障恢复）
  • 诊断并修复 Telegram API 的 TLS 代理兼容性问题（LibreSSL vs OpenSSL）
  • Anthropic API 余额耗尽 → 自动尝试 Gemini（区域限制）→ Groq（OAuth 被墙）→ 成功接入硅基流动 Qwen2.5
  • Bot 支持自然语言触发拍照、录视频、截屏并推送，兼具多轮对话能力
  • 全程 Node.js，自动处理代理隧道、文件上传、错误重试

系统级服务部署

开机自启 · 防休眠 · 自动重启
  • 配置 caffeinate -dims 防止 Mac 休眠，确保 Bot 24 小时在线
  • 编写 LaunchAgent plist，实现 caffeinate + Bot 双服务开机自动启动
  • KeepAlive=true 配置崩溃后自动重启，无需人工干预

05 · Insight

能力边界的重新定义

当 AI 遇到障碍（API 不可用、权限被拒、代理失效），它不是报告错误等待人工介入——而是自主分析根因、尝试替代路径、完成目标。这是与传统 Copilot 类工具最根本的差异。

这种用法与 Cursor/OpenClaw 面向开发者的代码协作模式形成互补：

  • Cursor 帮你写更好的代码
  • Claude MCP 帮你操控整台电脑

对于系统运维、IoT 控制、自动化工作流等场景，这种「AI 作为操作员」的范式有着显著的效率优势——尤其是在需要跨系统、跨服务、跨硬件协同的复杂任务中。

01 · Overview

Not AI Writing Code—
AI Operating the Whole Machine

This case records a single continuous conversation in which Claude AI was driven by plain natural-language instructions to complete camera capture and recording, full-stack Telegram Bot deployment, and system service auto-start—without manually writing or executing any code.

Core insight: Claude MCP (Model Context Protocol) extends AI’s reach from “text generation” to “physical-world execution.” Here the AI acts as system operator, not code advisor.

03 · What Was Built

Projects Completed in One Session

Multimedia hardware control

Camera · video · voice
  • Reset permissions via tccutil + AppleScript to bypass MCP process limits for live camera capture
  • Use ffmpeg avfoundation to record 1280×720 video and push to Telegram
  • Use macOS say for Chinese TTS (.m4a) and play through system speakers
  • Auto-detect frame-rate mismatch (30fps → 29.97fps) and fix without human input

Telegram Bot full-stack deployment

From zero to production (including failure recovery)
  • Diagnose and fix Telegram API TLS proxy issues (LibreSSL vs OpenSSL)
  • Anthropic balance exhausted → try Gemini (region blocked) → Groq (OAuth blocked) → succeed with SiliconFlow Qwen2.5
  • Bot supports natural-language trigger for photo, video, screenshot and push, plus multi-turn chat
  • Node.js throughout; proxy tunnel, file upload, retries handled automatically

System-level service deployment

Auto-start · no sleep · auto restart
  • caffeinate -dims to keep Mac awake so Bot stays online 24/7
  • LaunchAgent plist for caffeinate + Bot to start on boot
  • KeepAlive=true for automatic restart on crash

05 · Insight

Redrawing the boundary of what AI can do

When the AI hits obstacles (API down, permission denied, proxy failed), it doesn’t just report an error and wait—it analyzes cause, tries alternatives, and completes the goal. That’s the fundamental difference from traditional Copilot-style tools.

This complements Cursor/OpenClaw’s code-collaboration model:

  • Cursor helps you write better code
  • Claude MCP helps you operate the whole machine

For ops, IoT control, and automated workflows—especially when crossing systems, services, and hardware—“AI as operator” offers a clear efficiency gain.
